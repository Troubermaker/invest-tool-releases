# K 线技术分析指标使用指南

> 本工具内置的 6 项自动技术分析功能：压力/支撑、箱体、平台、趋势线、波段、通道。
> 本文说明各自原理、用法及组合策略。

---

## 速览

| 指标 | 视觉形式 | 颜色与线型 | 表达内容 |
|---|---|---|---|
| 压力/支撑 | 横向直线 | 红色虚线 / 绿色点线 | 近 N 根 K 线的最高/最低价位 |
| 箱体 | 半透明矩形 | 蓝紫色 + 虚线边 | 持续震荡区间 |
| 平台 | 半透明矩形 | 青绿色 + 实线边 | 缩量横盘的强势整理区 |
| 趋势线 | 斜线 | 蓝色实线 / 橙色虚线 | 上升支撑线 / 下降压力线 |
| 波段 | 折线骨架 + 节点圆点 | 蓝实线段 / 橙虚线段 | 价格主结构（去除小波动） |
| 通道 | 平行四边形 | 青色边 + 半透明填充 | 价格运行的"轨道"上下界 |

色弱友好原则：所有指标都同时通过**颜色 + 线型 + 形状**三重冗余编码。
- 蓝 = 上行 / 升势 / 支撑
- 橙 = 下行 / 跌势 / 压力
- 实线 = 升势相关，虚线 = 跌势相关
- 实心圆 = 高点，空心圆 = 低点

## 一、压力位 / 支撑位

### 原理

近 N 根 K 线的最高价 = 压力，最低价 = 支撑，各一条线。

| 周期 | N（窗口） |
|---|---|
| 日 K | 60 根（约 3 个月） |
| 周 K | 26 根（约半年） |
| 月 K | 12 根（一年） |
| 年 K | 5 根 |

**防"插针"过滤**：候选高/低必须在 ±2 根内有相邻 K 线在 ±1.5% 价差内"认可"，否则跳到次优候选。这避免了一次极端单日波动把价位"绑架"成压力/支撑。

### 实战用法

| 场景 | 解读 |
|---|---|
| 价格上涨触及压力线 | 可能反转或调整 |
| 价格下跌触及支撑线 | 可能反弹 |
| 突破压力 + 放量 | 上行延续信号 |
| 跌破支撑 + 放量 | 下行加速信号 |
| 现价已是窗口最高 | 不显示压力线（已在突破中） |
| 现价已是窗口最低 | 不显示支撑线（创新低中） |

### 局限

- 只看近期窗口，不算历史多次试探的"老压力"
- 单纯反映价格极值，不考虑成交量

## 二、箱体

### 原理

滑窗扫描，连续 ≥8 根 K 线满足 `(最高 - 最低) / 中位 ≤ 12%` → 视为震荡区。
找到候选后贪心向后扩展，直至破位或达到 80 根上限。

### 实战用法

| 场景 | 操作思路 |
|---|---|
| 价格在箱内 | 高抛低吸，贴近上沿减仓、贴近下沿加仓 |
| 突破箱顶 + 放量 | 突破信号，目标价 ≈ 箱顶 + 箱高 |
| 跌破箱底 + 放量 | 破位信号，下行参考 ≈ 箱底 - 箱高 |
| 长期窄幅箱体 | 越窄越久 → 突破后能量越强 |

### 局限

- 12% 阈值偏宽松，部分箱体跨度大
- 真正"高质量"的横盘看下一项「平台」更可靠

---

## 三、平台

### 原理

平台是「箱体的加强版」，必须同时满足：

1. 时间足够长：≥ 20 根
2. 振幅足够窄：≤ 10%
3. **量能确认（核心）**：平台内均量 / 平台前 30 根均量 ≤ 0.85（缩量 ≥ 15%）

### 实战用法

平台代表"价格横盘 + 资金观望或暗中吸筹"，比纯箱体更有意义：

- **平台越长 + 缩量越明显 → 突破后爆发力越强**
- 突破方向 = 资金最终选择方向，**带量突破**比无量突破可靠得多
- 突破后参考"平台高度"作为目标价

### 局限

- 需要量数据，少数股票（如部分北证票）量数据不全可能漏检
- 只能确认"缩量"，不能识别资金性质（机构/游资/散户）

## 四、趋势线

### 原理

1. 滑动窗口（±5 根）找 swing 高/低点
2. 同类型 swing 两两连线生成候选
3. 校验：从起点到当前 K 线，无任何 bar 突破/跌破该线（容差 ±1.2%）
4. 触及次数 ≥ 2、未破位的按"触及数 × 时间跨度"打分
5. 上升趋势线（连低点）和下降趋势线（连高点）各取最强 1 条

### 视觉

- **蓝色实线**（连低点 = 上升趋势线 = 支撑）
- **橙色虚线**（连高点 = 下降趋势线 = 压力）

### 实战用法

**蓝色上升支撑线：**
- 价格回踩到线 → 潜在买点（趋势未破 + 回调到位）
- 有效跌破（带量收盘穿破）→ 上升趋势告破

**橙色下降压力线：**
- 价格反弹到线 → 潜在卖点 / 减仓
- 有效突破（带量站上）→ 下降趋势可能反转
- **突破后压力线常反转成支撑**（量能确认时尤其有效）

**两条都出现** = 三角形整理，价格在两线之间运行，收敛终将选择方向。

### 局限

- 一旦显示出来又被突破 → 算法不会预警，需下次重算才更新
- 趋势线越斜越脆弱，缓和的趋势线持续性更好

## 五、波段（Zigzag）

### 原理

1. 大窗口（±10 根）找 swing 高/低，过滤小噪声
2. 高低交替化：连续两个同类型 swing 只保留更极端的（连续高保留更高，连续低保留更低）
3. 输出严格"高→低→高→低"交替的折线节点

### 视觉

折线分段着色：

- **上行段**（low → high）：蓝色实线
- **下行段**（high → low）：橙色虚线
- 节点圆点：高点 = 橙色实心，低点 = 白底蓝边

### 实战用法

| 用途 | 做法 |
|---|---|
| 看清主结构 | 去除小波动后整体走势在干嘛 |
| 确认波段方向 | 当前在第几波，正在涨还是跌 |
| 测算波段空间 | 历史每段涨跌幅可作为下一段的参考 |
| 识别背离 | 连续创新高但 MACD/KDJ 走低 = 顶背离信号 |

### 局限

- **最右一段未确认**：最新极值未经过 N 根 K 线验证，可能后续被刷新
- pivotWindow 决定灵敏度（默认 10）：调大 → 只看大波段；调小 → 包含小波动

## 六、趋势通道（Raff 平行通道）

### 原理

1. 取最近 zigzag 的 swing 高 + swing 低
2. 分别用最小二乘线性回归得到两条 slope，**取均值**作为通道统一斜率
3. 上轨截距 = `max(swing_high.price - slope × idx)` → 包住所有 swing 高
4. 下轨截距 = `min(swing_low.price - slope × idx)` → 包住所有 swing 低
5. 上下轨**严格平行**，跟手画"趋势线 + 平行复制"一致

### 视觉

青色 cyan 上下边线 + 半透明青色填充的平行四边形。

### 实战用法

| 价格位置 | 含义 |
|---|---|
| 贴上轨 | 高位区，潜在卖点 / 减仓机会 |
| 贴下轨 | 低位区，潜在买点 / 加仓机会 |
| 在中间 | 中性，等待方向选择 |
| 突破上轨 | 趋势加速向上（带量更可靠） |
| 跌破下轨 | 趋势加速向下（带量更可靠） |

通道斜率 = 主趋势的方向与速度：
- 斜率正且陡 → 强势上涨
- 斜率正且缓 → 健康慢牛
- 斜率近水平 → 横盘震荡
- 斜率负 → 下跌通道

### 局限

- 必须有 zigzag swing 数据，新股或极端行情可能不显示
- 强制平行 → 喇叭口/楔形格局下不太贴合（那种走势本来也不该叫通道）

## 组合用法（最重要的部分）

### 三种典型行情匹配指标

#### 1. 趋势行情（单边上升或下跌）

主力工具：**趋势线 + 通道**
辅助工具：**波段**（确认分波节奏）
不必使用：箱体 / 平台（无横盘）

#### 2. 横盘整理

主力工具：**箱体 + 平台**
辅助工具：**压力/支撑位**（确认上下边界）
不太适用：趋势线 / 通道（无明确方向）

#### 3. 转折期（不确定）

主力工具：**波段**（看是否打破前一波结构）
辅助工具：**趋势线 + 压力/支撑位**（共同验证）

---

### 推荐组合（按场景）

#### 个人复盘标配
- 波段（zigzag）—— 看整体结构
- 压力/支撑 —— 看关键位
- 其他作为辅助按需打开

#### 寻找突破机会
- 平台 —— 锁定"缩量横盘 + 时间足够"的票
- 通道 —— 看突破方向
- 压力/支撑 —— 看突破后的上方空间

#### 跟踪持仓
- 趋势线 —— 确认主趋势是否还在
- 波段 —— 看回调/反弹是否到位
- 通道 —— 看是否触及上沿（减仓）或下沿（补仓）

---

### 多重信号共振（核心组合）

多个独立指标在同一价位给出相同方向的信号 = "共振"，可信度远高于单一指标。

| 多重信号叠加 | 解读 |
|---|---|
| 价格在压力位 **+** 通道上轨 **+** 下降趋势线 | 三重压力共振，反转概率高 |
| 价格在支撑位 **+** 通道下轨 **+** 上升趋势线 | 三重支撑共振，反弹概率高 |
| 平台突破 **+** 突破压力位 **+** 站上通道上轨 | 突破有效，可跟进 |
| 跌破支撑 **+** 跌破上升趋势线 **+** 跌破通道下轨 | 趋势恶化，止损或离场 |
| 波段最高点 **+** 通道上轨 **+** 缩量 | 顶部信号 |
| 波段最低点 **+** 通道下轨 **+** 缩量 | 底部信号 |

### 信号可信度排序（弱 → 强）

1. **单一信号**：任何一个独立指标的信号
2. **同类共振**：两条压力/支撑位对齐
3. **多类共振**：趋势线 + 通道 + 压力支撑同方向
4. **多类共振 + 量能确认**：缩量平台 + 突破压力 + 放量

## 参数调整指南

### 不同周期的合理设置

代码层面可调（默认值已经针对日 K 优化）：

| 参数 | 日 K | 周 K | 月 K |
|---|---|---|---|
| swing pivotWindow | 5 ~ 10 | 3 ~ 5 | 2 ~ 3 |
| 通道 maxPivots | 5 | 4 | 3 |
| 箱体 minBars | 12 ~ 20 | 8 ~ 12 | 6 ~ 8 |
| 平台 minBars | 20 ~ 30 | 12 ~ 20 | 8 ~ 12 |

### 不同股票特性

| 股票类型 | 调参方向 |
|---|---|
| 大盘股 / 蓝筹 | 波动小，箱体阈值收紧（8% 而非 12%） |
| 中小盘 / 题材 | 波动大，阈值放宽（15% ~ 20%） |
| 次新股 | 上市时间短，所有"近 N 根"参数缩短 |
| 长期慢牛 | 通道斜率小，pivotWindow 调大（看大波段） |
| 暴涨暴跌 | pivotWindow 调小，及时识别新结构 |

## 局限与注意事项

1. **算法基于历史数据** —— 不能预测未来，只能识别已经形成的形态
2. **A 股特殊性** —— 涨跌停制度、消息驱动会让纯技术分析失灵
3. **复盘工具，不是交易系统** —— 决策应结合基本面、消息面、资金面
4. **算法参数没有"最优值"** —— 不同股票不同周期可能需要微调
5. **多指标共振 ≠ 必然方向** —— 仍可能反向，但概率上更可靠
