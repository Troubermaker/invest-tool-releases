# Invest Tool 使用指南

面向**已部署完毕**的用户。开发环境搭建见 [README.md](./README.md)。

---

## 整体工作流

应用核心是一条「**发现 → 收藏 → 追踪**」流水线：

```
量化选股（发现） ──★收藏──→ 候选池（追踪） ──◯结仓──→ 持仓 / 复盘
                                ↓
                         （也可以 +自选 进自选池）
```

- **量化选股**：跑 detector 全市场扫描，找出符合形态的票
- **候选池**：从扫描结果里你主动 ⭐ 选中的票，持续刷新追踪
- **自选**：跟候选池正交，长期关注但不一定是技术形态命中的票

## 量化选股（admin 专属）

侧边栏「选股」进入。三个扫描器，每个 tab 一种策略，**互不冲突可叠加**。

### 1. 近期发车

> **抓**：detector 找到的「最近 N 天 stage 3 突破」 + 还在持仓窗口内的票

底层：`detectThreeStageLaunch` + `backtestStock`，5 维评级（`gradeFreshSignal`）+ 周线趋势确认。

工具栏：
- **突破时段** chips（≤1d / ≤3d / ≤5d / ≤7d）：限制 `holdBars`，越小越新鲜
- **数据完整度看板**：显示「X / 全市场 已是 YYYY-MM-DD · 缺 N」状态
- **下载今日 K**：按通达信式增量更新，只下载缺数据的票
- **重新扫描** / 取消

表格列：评级 / 周共振 / 股票 / 突破日 / 持有天数 / 入场价 / 当前价 / 收益 / 操作

排序：**周共振 ✓ > grade B > A > C > S > 收益降序**。
- B 是最推荐档（红 chip），S 是透支警示档（紫 chip）
- 周共振 ✓ 的票胜率高 ~8pp

### 2. 龙回头

> **抓**：曾涨停拉升过的龙头，回踩 30% 左右后开始二次启动的票

底层：`detectDragonReturn`。五道关 AND：

| 关 | 条件 |
|---|------|
| 龙的识别 | 15 根内 ≥3 涨停（含 3 天内 2 涨停的紧密簇）|
| 高点位置 | 高点必须 ≤ 15 根前 |
| 回踩深度 | 高点回踩 15-30%（太浅没回到位 / 太深结构破坏）|
| 企稳收敛 | 近 5 根振幅 ≤ 10% |
| 启动信号 | 近 5 根任一根：收阳 + 量比 ≥ 1.7× 地量基期 + 实体 ≥50% 或涨幅 ≥4% |

表格列：股票 / 涨停板数 / 高点 / 回踩% / 启动日 / 量比 / 现价 / 止损 / 操作

排序：涨停数降 → 启动信号靠后（更新鲜）

### 3. 连板游资（Ptrade 1:1 迁移）

> **抓**：今日触板 + 妖股基因的票，给 T+1 短打

底层：`detectLimitUpRelay`，照搬 Ptrade 量化平台「极速优化版游资连板策略」。

入选条件：
- 板块：仅**主板 + 中小板**（60/00 开头），剔除 ST / 科创（688）/ 创业（300）/ 北交所
- 上市 ≥ 60 个交易日（剔除次新）
- 今日触板（涨停 1.10 / 创业科创 1.20 / ST 5%，含成交量约束）
- 三种形态任一：
  - **A 连板接力**：今天涨停 + 昨天涨停
  - **B 反包确认**：今天涨停 + 昨天未涨停 + 昨日跌幅 < 8%
  - **C 摸板试盘**：今天 high 触涨停但收盘未封 + 收涨 + 上影线 < 5%
- 妖股基因：
  - 近 5 个**有效交易日**（排除停牌）≥ 2 涨停
  - 现价 ≥ 近 20 日最高 × 0.95

表格列：股票 / 形态 / 连板（首板/N连板/断板）/ 5天板数 / 现价 / 涨停价 / 操作

排序：连板高度降 → 5 天板数降

## 候选池

侧边栏「候选」进入。**已收藏的票**做持续追踪。

### Tab 切换

- **全部**：所有候选
- **候选**：source 含「候选」的（多为 stage 1/2 蓄势期）
- **发车**：source 含「发车」的（多为 stage 3 已突破）

### 双重筛选 + 多源命中

每只票都带**来源标签**（彩色 chip），表示它来自哪个扫描器：
| 标签 | 颜色 | 来源 |
|------|------|------|
| 三维启动 | 黄 | 找候选 / 找发车扫描器 |
| 近期发车 | 青 | Quant → 近期发车 |
| 龙回头 | 红 | Quant → 龙回头 |
| 连板游资 | 橙 | Quant → 连板游资 |
| 其它 | 灰 | 自定义 / 旧数据 |

**多源命中**：同一只票被多个扫描器选中时，会显示**多个 chip 叠加**（如「龙回头 + 连板游资」），是质量信号 —— 多个 detector 同时认可，可信度更高。

工具栏的 chip 行：
```
[全部] [临门一脚] [已突破] ... | [全部来源] [龙回头 5] [连板游资 3] ...
        ↑ 状态筛选            ↑ 来源筛选（同时点选可叠加）
```

### 状态语义（lazy 计算）

| 状态 | 触发 | 颜色 |
|------|------|------|
| 等待中 | 现价 ≤ 入选价的 break_level - 1.5% | 灰 |
| 临门一脚 | 现价距突破点位 ≤ 1.5% | 红 |
| 已突破 | 现价 ≥ break_level | 红粗 |
| 假突破回落 | 突破后又跌回 break_level 之下 5% 内 | 警示 |
| 进入买点 | 现价 ±2% 黄金买点（s2Mid）| 红 |
| 已失效 | 现价跌破 s1_lower（蓄势下沿）| 灰删除线 |

### 列拖拽自定义

中间 12 列可以自由拖动重排，顺序自动保存到 `user_preferences`，下次打开应用还原。

固定列（不参与拖动）：☑ checkbox / 股票 / 操作

### 刷新追踪

工具栏「刷新候选池」按钮触发：
1. 对每只候选拉日 K（缓存命中秒返）
2. 重跑 `detectThreeStageLaunch` 评估当前形态状态（`formation_state`）
3. 算入选以来的最大涨幅（`peak_gain_since_save`）
4. 检测二次买点（`secondary_entry_at`）和突破日（`breakout_at`）

#### 自动剔除逻辑

刷新时同时清理「已死」的票：

| 触发 | 行为 |
|------|------|
| Stage 3 突破 > 40 根仍无新结构 | 视为「过期龙」，从候选池移除 |
| 形态作废（跌破蓄势下沿） > 5 根 | 视为「死龙」，从候选池移除 |
| Stage 3 突破 ≤ 40 根但已 exhausted | 保留观察，不剔除 |

刷新结果会提示「更新 N · 剔除过期 M」。

## 数据缓存与 TDX

### 缓存机制

K 线数据存在 `kline_cache.db`（独立于 `invest_data.db`），WAL 模式支持并发读。

按交易日命中：
- 周末 / 节假日：cached_date >= 节前最后交易日 → 直接命中，**不调 TDX**
- 工作日盘前：cached_date >= 昨日 → 命中
- 工作日盘后（≥ 15:00）：cached_date == today + last K = today → 命中
- 否则：跨日增量拉取（gap+5 根，merge）

### 下载今日 K

「近期发车」工具栏按钮，**仅在收盘后建议使用**。

预检 → 弹确认对话框 → 顺序下载（10 并发批 + 自适应 gap）。
- 缓存命中的票毫秒级跳过（不调 TDX）
- 缺数据的拉增量（gap+5 根）
- 失败的退回全量重拉一次

成功判定**严格**：必须 last K 日期 ≥ target_date 才算成功。即使 `data.length > 0` 但 last K 是昨天的话，会计入「失败」（防止 TDX 静默返老缓存被误判为成功）。

### A 股代码列表缓存

同样按交易日命中，磁盘文件 `kline_cache/a_share_codes.json`：
- 内存命中（同进程同日）→ 秒返
- 磁盘命中（cached_date >= 最近交易日）→ 不调 TDX
- TDX 失败时兜底用磁盘旧缓存（带警告）

周末打开应用扫描，**完全不会调 TDX**，全部缓存命中。

### 日志

如果发现扫描慢或拉数据失败：
- 文件日志：`logs/tdx.log`（5MB × 3 滚动）
- 关键字：
  - `拿到 N 根` → 成功拉取
  - `两个服务器都返空` → TDX 服务器侧问题
  - `增量返空，退回全量重拉` → 自动 fallback 触发
  - `命中磁盘缓存` → 没调 TDX
  - `异常：xxxx` → 网络 / 连接问题

## 自选

侧边栏「自选」。跟候选池正交：
- 没有 snapshot / 形态状态约束，纯粹是「长期关注列表」
- 支持分组管理 + 实时报价（3 秒刷新）
- 可以从任意 view 用「+ 自选」按钮添加

跟候选池的关系：
- 候选池里的票可以同时也在自选
- 自选的票可以单独存在不进候选池
- 「+ 自选」按钮跟「☆ 收藏到候选」是两个独立动作

---

## 持仓

侧边栏「持仓」。已建仓的票：
- 自动算成本 / 浮盈 / 仓位占比

## 常见问题

### 选股 tab 选不到票

**先看是哪个扫描器**：
- 近期发车：调宽「突破时段」（≤5d / ≤7d）
- 龙回头：市场近期可能确实没合格龙头，可以等几天
- 连板游资：连板要求是硬约束，没有满足条件的票就是没有

### 数据明明已收盘还显示「缺 N」

可能是 TDX 服务器还没推今日 K。**等 30 分钟再点下载**，多数情况会解决。

如果反复点下载都失败：检查 `logs/tdx.log`。如果都是「两个服务器都返空」，那就只能等。

### 候选池的票一直显示「未识别」

需要点工具栏的「刷新候选池」按钮触发 detector 评估。第一次刷新可能慢（每只票单独拉日 K），之后缓存命中会快很多。

### 列拖拽顺序丢失

顺序保存在 `user_preferences` 表里。如果重置数据库会丢失；正常使用情况下永久保留。
