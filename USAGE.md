面向**已部署完毕**的用户。开发环境搭建见 [README.md](./README.md)。

## 实战决策清单（基于 368 笔回测 + 4 季度跨段验证）

每天早盘 / 周末复盘走这个流程：

```
1. 看顶部 regime badge
   - 强势/良好 → 进入主升突破 tab
   - 弱势 → 主升突破收紧后仍可看；优先看突破前夜（埋伏）
   - 破位 → 仅看突破前夜底部埋伏

2. 主升突破 tab → 按星级取信号
   ⭐⭐⭐ medium confirm   主力仓位（76% 胜率，跨段稳定）
   ⭐⭐ strong confirm    机会仓位（牛市 91% / 震荡市 40%，看 regime）
   ⭐ ML / sector ≥70     备选参考（55-60% 胜率）

3. 突破前夜 tab（周末复盘）→ 给质量 ≥70 / MACD ≥50 的票设价格预警
   突破触发后会自动出现在主升突破 tab

4. 龙回头 tab → 位置 ✓ + 探底 ⛏ + 浅回踩（12-18%）= 优选
5. 连板游资 tab → 弱势市自动暂停；强势市可短打 1-3 天
```

## 整体工作流

应用核心是一条「**发现 → 收藏 → 追踪**」流水线：

```
量化选股（发现） ──★收藏──→ 候选池（追踪） ──◯结仓──→ 持仓 / 复盘
                                ↓
                         （也可以 +自选 进自选池）
```

- **量化选股**：跑 detector 全市场扫描，找出符合形态的票
- **候选池**：从扫描结果里你主动 ⭐ 选中的票，持续刷新追踪
- **自选**：跟候选池正交，长期关注但不一定是技术形态命中的票

## 量化选股（admin 专属）

侧边栏「选股」进入。4 个 tab，**互不冲突可叠加**。

> 顶部右侧 **大盘 regime 徽章**（强势/良好/震荡/弱势/破位）会自适应调整 detector 阈值。
> 弱势 / 破位市场会自动暂停部分策略 + 收紧阈值。

### 1. 主升突破（旧名「近期发车」）

> **抓**：detector 找到的「最近 N 天 stage 3 突破」+ 还在持仓窗口内的票

底层：`detectThreeStageLaunch` + `backtestStock` + Week 1-4 多层过滤。

**⭐ 星级体系**（基于 4 季度跨段验证 n=368，2024-10 ~ 2026-05）：

| 星级 | 触发条件 | 4 段平均胜率 | 跨段范围 | 月信号 | 建议 |
|---|---|---|---|---|---|
| **⭐⭐⭐** | `breakoutConfirm = 'medium'`（N+1 平开收阳 + 量能维持）| **76%** | **62-91%** | 4-6/月 | **主力仓位** |
| ⭐⭐ | `breakoutConfirm = 'strong'`（N+1 高开≥1% + 不破中点）| 63% | 40-91% | 3-4/月 | 看大盘 regime |
| ⭐ | sectorStrength ≥ 70 / mlScore ≥ 0.40 | 55-60% | 50-62% | 10+/月 | 参考 |

**反直觉但关键的发现**：
- `strong` 信号 regime-specific —— 牛市 91% 胜率，震荡市 40%，**单用风险高**
- `medium` 反而最稳 —— 4 季度都 ≥61%，散户进场成本低 + 上行空间足
- ML 模型 AUC 仅 0.55，**作为辅助而非主力**

**工具栏**：
- 突破时段 chips（≤1d / ≤3d / ≤5d / ≤7d）
- 数据完整度看板
- 下载今日 K（仅日K，跟扫描范围严格对齐）
- 重新扫描 / 取消

**表格列**：ML/星级 / 板块 / 评级 / 周 / 股票 / 突破日 / 持有天数 / 入场价 / 当前价 / 收益 / 操作

**排序**（综合评分）：
1. 星级强排序（⭐⭐⭐ > ⭐⭐ > ⭐ > 普通）
2. 同档内：composite = ML×0.5 + sector×0.3 + weekly×0.2
3. 最终：returnPct 降

### 2. 突破前夜（新增）

> **抓**：还没突破但临门一脚的票（Stage 1 蓄势中 / Stage 2 试盘后）

底层：`detectThreeStageLaunch` Stage 1/2 + 蓄势质量评分 + MACD 金叉预兆。

**入选条件**：
- 仍处 stage 1（蓄势）/ stage 2（试盘）
- 距 `s1Upper` 突破位 ≤ 5%（regime 自适应：强势市 ≤8% / 弱势 ≤3%）
- 蓄势 ≤ 80 根（避免长期卡死的票）

**质量评分**（4 维 × 25 分 = 0-100）：
- 振幅收窄（后半段 / 前半段 < 0.85）
- 量能萎缩（后半段均量 / 前半段 < 0.7）
- MA 粘合（MA5/10/20 spread < 2%）
- 主力潜伏（突破前 10 天温和放量 2-4 根）

**MACD 金叉预兆**（0-100）：DIF/DEA gap 在收敛 + 还没金叉 + 上行趋势 = 高分

**表格列**：阶段 / **质量** / **MACD** / 股票 / 现价 / 突破位 / 距突破 / 蓄势天 / 止损位 / 操作

**用法**：周末复盘扫一遍，给质量 ≥70 / MACD ≥50 的票设价格预警 = `s1Upper`，等突破触发自动通知。

### 3. 龙回头

> **抓**：曾涨停拉升过的龙头，回踩 15-30% 后二次启动的票

底层：`detectDragonReturn` + Week 3 深度优化。

**核心五关**：

| 关 | 条件 |
|---|------|
| 龙的识别 | 15 根内 ≥3 涨停（含 3 天内 2 涨停紧密簇）|
| 高点位置 | 高点必须 ≤ 15 根前 |
| 回踩深度 | 15-30%（甜区 12-18% → quality 100）|
| 企稳收敛 | 近 5 根振幅 ≤ 10% |
| 启动信号 | 近 5 根任一：收阳 + 量比 ≥1.7× + 实体 ≥50% 或涨幅 ≥4% |

**Week 3 新增过滤**：
- **位置因子 ✓**：现价 > MA200 + 距 60 日新高 < 30%（防长跌反弹陷阱）
- **探底试盘 K ⛏**：启动前 10 天里出现长下影 + 站稳的探底成功信号
- **回踩深度甜区**：12-18% 强势浅回踩=100 分；18-25%=70；25-30%=40

**表格列**：股票 / 涨停 / 高点 / 回踩(色码) / **位置** / **探底** / 启动日 / 量比 / 现价 / 止损 / 操作

**排序**（综合评分）：pullbackQuality×0.40 + probe×0.20 + position×0.15 + limitUps×0.15 + sector×0.10

### 4. 连板游资（Ptrade 1:1 迁移）

> **抓**：今日触板 + 妖股基因的票，给 T+1 短打

底层：`detectLimitUpRelay`，照搬 Ptrade 量化平台「极速优化版游资连板策略」。

入选条件：
- 板块：仅**主板 + 中小板**（60/00 开头），剔除 ST / 科创（688）/ 创业（300）/ 北交所
- 上市 ≥ 60 个交易日（剔除次新）
- 今日触板（涨停 1.10 / 创业科创 1.20 / ST 5%，含成交量约束）
- 三种形态任一：
  - **A 连板接力**：今天涨停 + 昨天涨停
  - **B 反包确认**：今天涨停 + 昨天未涨停 + 昨日跌幅 < 8%
  - **C 摸板试盘**：今天 high 触涨停但收盘未封 + 收涨 + 上影线 < 5%
- 妖股基因：
  - 近 5 个**有效交易日**（排除停牌）≥ 2 涨停
  - 现价 ≥ 近 20 日最高 × 0.95

表格列：股票 / 形态 / 连板（首板/N连板/断板）/ 5天板数 / 现价 / 涨停价 / 操作

排序：连板高度降 → 5 天板数降

## 智能层：大盘 regime + 板块强度 + ML 模型

3 层环境感知，作为所有策略的前置过滤 + 信号增强。

### 大盘 regime（5 档自适应）

顶部右侧徽章实时显示。基于 4 维评分（沪深 300 趋势 50% + 市场宽度 20% + 创业板相对强度 15% + 量能 15%）：

| Regime | 评分 | 策略动作 |
|---|---|---|
| 🔥 强势市 | ≥75 | 全策略放宽阈值，多抓信号 |
| ✓ 良好市 | 60-75 | 默认参数 |
| — 震荡市 | 40-60 | 默认参数 |
| ⚠ 弱势市 | 25-40 | 主升突破/龙回头收紧 + 强制 N+1 确认；突破前夜仍可埋伏 |
| ✗ 破位市 | <25 | 主升突破/龙回头/连板游资 **全部暂停**；仅突破前夜可抄底 |

### 板块强度（0-100）

底层 KPL 数据 + 排名分位 + 主力净流入复合评分。

**实测胜率提升 +10pp**（>=70 板块的票 vs 非 top 板块）。**但 4 段稳定性 12pp**，作为辅助筛选可靠。

4 个 tab 表格都有「板块」列显示，红色加粗代表 ≥70 强势板块。

### ML 模型（Phase 3）

LightGBM 二分类器，预测每个突破信号的 T+7 盈利概率（0-1）。

**真实 holdout AUC = 0.55**（弱信号），**作为辅助而非核心** —— 主信号来自规则层（breakoutConfirm）。

模型自动加载 `models/signal_filter_phase3only_*.pkl` 最新版本。

重训流程：
```bash
# 1. 在 console 跑攒数据
await bt.runDump({ name: 'next_v1' })

# 2. 训练（独立 venv）
pip install -r requirements-ml.txt
python -m services.ml_signal_filter
```

## 候选池

侧边栏「候选」进入。**已收藏的票**做持续追踪。

### Tab 切换

- **全部**：所有候选
- **候选**：source 含「候选」的（多为 stage 1/2 蓄势期）
- **发车**：source 含「发车」的（多为 stage 3 已突破）

### 双重筛选 + 多源命中

每只票都带**来源标签**（彩色 chip）：
| 标签 | 颜色 | 来源 |
|------|------|------|
| 主升突破 | 青 | Quant → 主升突破（旧名"近期发车"，旧 source 字段兼容）|
| 突破前夜 | 黄 | Quant → 突破前夜（旧名"找候选"，旧 source 字段兼容）|
| 龙回头 | 红 | Quant → 龙回头 |
| 连板游资 | 橙 | Quant → 连板游资 |
| 其它 | 灰 | 自定义 / 旧数据 |

**多源命中**：同一只票被多个扫描器选中时，会显示**多个 chip 叠加**（如「龙回头 + 连板游资」），是质量信号 —— 多个 detector 同时认可，可信度更高。

工具栏的 chip 行：
```
[全部] [临门一脚] [已突破] ... | [全部来源] [龙回头 5] [连板游资 3] ...
        ↑ 状态筛选            ↑ 来源筛选（同时点选可叠加）
```

### 状态语义（lazy 计算）

| 状态 | 触发 | 颜色 |
|------|------|------|
| 等待中 | 现价 ≤ 入选价的 break_level - 1.5% | 灰 |
| 临门一脚 | 现价距突破点位 ≤ 1.5% | 红 |
| 已突破 | 现价 ≥ break_level | 红粗 |
| 假突破回落 | 突破后又跌回 break_level 之下 5% 内 | 警示 |
| 进入买点 | 现价 ±2% 黄金买点（s2Mid）| 红 |
| 已失效 | 现价跌破 s1_lower（蓄势下沿）| 灰删除线 |

### 形态识别（detector 写入 formation_state）

候选池里每只票刷新时会跑两个 detector，按优先级回退：

| formation_state | 来源 detector | 含义 | 表格徽章 |
|----|------|------|------|
| `consolidating` | detectThreeStageLaunch | S1 蓄势中 | 绿 ✓三维 |
| `tested` | detectThreeStageLaunch | S2 试盘后 | 绿 ✓三维 |
| `breakout` | detectThreeStageLaunch | S3 刚突破（≤5 根）| **红 ✓三维** |
| `rally` | detectThreeStageLaunch | S3 主升中 | **红 ✓三维** |
| **`stretched`** | **detectStretchedRally** | **长横盘 → 直接跳空突破**（无 S2 试盘环节）| **红 🚀 跳空** |
| `exhausted` | detectRallyExhaustion | 衰竭警示 | 无 |
| `invalid` | detectRallyExhaustion | 形态作废 | 无 |
| `unknown` | — | 未刷新或检测失败 | 无 |

**两个 detector 互补**：
- detectThreeStageLaunch：经典三段式（蓄势 + 试盘 + 突破）
- detectStretchedRally：横盘跳空型（蓄势 + 直接跳空，不要求试盘 K）

**互补触发优先级**：
1. 优先 detectThreeStageLaunch
2. 三维到 S1/S2 但没确认 S3 → 试 stretched，命中即用
3. 三维没找到任何事件 → 试 stretched 兜底
4. 都没找到 → unknown

也就是说**已经在 S3 主升的票（breakout/rally）不会被 stretched 覆盖**，stretched 只在三维 detector 看不到突破的情况下补位。

### 列拖拽自定义

中间 12 列可以自由拖动重排，顺序自动保存到 `user_preferences`，下次打开应用还原。

固定列（不参与拖动）：☑ checkbox / 股票 / 操作

### 刷新追踪

工具栏「刷新候选池」按钮触发：
1. 对每只候选拉日 K（缓存命中秒返）
2. 重跑 `detectThreeStageLaunch` 评估当前形态状态（`formation_state`）
3. 算入选以来的最大涨幅（`peak_gain_since_save`）
4. 检测二次买点（`secondary_entry_at`）和突破日（`breakout_at`）

#### 自动剔除逻辑

刷新时同时清理「已死」的票：

| 触发 | 行为 |
|------|------|
| Stage 3 突破 > 40 根仍无新结构 | 视为「过期龙」，从候选池移除 |
| 形态作废（跌破蓄势下沿） > 5 根 | 视为「死龙」，从候选池移除 |
| Stage 3 突破 ≤ 40 根但已 exhausted | 保留观察，不剔除 |

刷新结果会提示「更新 N · 剔除过期 M」。

## 交易日志（实战账本）

侧边栏「日志」进入。跟候选池正交：**候选池是 paper trading 观察哨，日志是真实仓位账本**。

### 核心价值

| 问题 | 解决方式 |
|---|---|
| 系统说 76% 胜率，实盘到底多少？ | 日志 vs 预期对照（顶部 "vs 预期 76%" 卡片）|
| ⭐⭐⭐ 信号实战表现 vs ⭐⭐ 差多少？ | 按 signal_source / star_level 切片 |
| 哪个 regime 我执行最好？ | 备注栏记录 + 后续归因 |

### 一键记录买入

主升突破 / 突破前夜 / 龙回头 / 连板游资 表格的操作列有 **📝** 按钮 → 一键加日志：
- 自动带入 entry_price / breakoutConfirm / sectorScore / ML / 等元数据快照
- 跳到日志页人工修正仓位 % / 止盈位

### 仓位建议（保守规则）

| 星级 | 推荐单笔 % | 最大单笔 % |
|---|---|---|
| ⭐⭐⭐ | 15% | 15% |
| ⭐⭐ | 10% | 15% |
| ⭐ | 5% | 10% |
| 普通 | 不推荐 | — |

**硬约束**：
- 同期持仓 ≤ 5 只
- 单板块仓位 ≤ 30%

### 平仓

行内"平仓"按钮 → 弹窗输入卖出价 + 原因（止盈/止损/时间/作废/手动） + 备注。系统自动算 pnl_pct + 持仓天数。

### 顶部统计卡片

- 总笔数 / 实际胜率 / vs 预期 76%（差 pp）/ 平均盈亏 / 最大单笔
- 各 signal_source 切片（主升突破 X 笔 Y% 胜率 / 龙回头 X 笔 Y% / ...）

### 关键警示

- **样本 < 30 笔时数据噪音大**，胜率波动 ±15pp 都正常
- **持续 1-2 个月再做归因决策**
- 实际胜率 vs 预期偏差大于 -15pp 时，触发 ML 重训（数据 regime 切换）

## 数据缓存与 TDX

### 缓存机制

K 线数据存在 `kline_cache.db`（独立于 `invest_data.db`），WAL 模式支持并发读。

按交易日命中：
- 周末 / 节假日：cached_date >= 节前最后交易日 → 直接命中，**不调 TDX**
- 工作日盘前：cached_date >= 昨日 → 命中
- 工作日盘后（≥ 15:00）：cached_date == today + last K = today → 命中
- 否则：跨日增量拉取（gap+5 根，merge）

### 下载今日 K

「近期发车」工具栏按钮，**仅在收盘后建议使用**。

预检 → 弹确认对话框 → 顺序下载（10 并发批 + 自适应 gap）。
- 缓存命中的票毫秒级跳过（不调 TDX）
- 缺数据的拉增量（gap+5 根）
- 失败的退回全量重拉一次

成功判定**严格**：必须 last K 日期 ≥ target_date 才算成功。即使 `data.length > 0` 但 last K 是昨天的话，会计入「失败」（防止 TDX 静默返老缓存被误判为成功）。

### A 股代码列表缓存

同样按交易日命中，磁盘文件 `kline_cache/a_share_codes.json`：
- 内存命中（同进程同日）→ 秒返
- 磁盘命中（cached_date >= 最近交易日）→ 不调 TDX
- TDX 失败时兜底用磁盘旧缓存（带警告）

周末打开应用扫描，**完全不会调 TDX**，全部缓存命中。

### 日志

如果发现扫描慢或拉数据失败：
- 文件日志：`logs/tdx.log`（5MB × 3 滚动）
- 关键字：
  - `拿到 N 根` → 成功拉取
  - `两个服务器都返空` → TDX 服务器侧问题
  - `增量返空，退回全量重拉` → 自动 fallback 触发
  - `命中磁盘缓存` → 没调 TDX
  - `异常：xxxx` → 网络 / 连接问题

## 自选

侧边栏「自选」。跟候选池正交：
- 没有 snapshot / 形态状态约束，纯粹是「长期关注列表」
- 支持分组管理 + 实时报价（3 秒刷新）
- 可以从任意 view 用「+ 自选」按钮添加

跟候选池的关系：
- 候选池里的票可以同时也在自选
- 自选的票可以单独存在不进候选池
- 「+ 自选」按钮跟「☆ 收藏到候选」是两个独立动作

## 常见问题

### 选股 tab 选不到票

**先看是哪个扫描器**：
- 近期发车：调宽「突破时段」（≤5d / ≤7d）
- 龙回头：市场近期可能确实没合格龙头，可以等几天
- 连板游资：连板要求是硬约束，没有满足条件的票就是没有

### 数据明明已收盘还显示「缺 N」

可能是 TDX 服务器还没推今日 K。**等 30 分钟再点下载**，多数情况会解决。

如果反复点下载都失败：检查 `logs/tdx.log`。如果都是「两个服务器都返空」，那就只能等。

### 候选池的票一直显示「未识别」

需要点工具栏的「刷新候选池」按钮触发 detector 评估。第一次刷新可能慢（每只票单独拉日 K），之后缓存命中会快很多。

### 列拖拽顺序丢失

顺序保存在 `user_preferences` 表里。如果重置数据库会丢失；正常使用情况下永久保留。
